# plans.json Schema

Auto-generated by `/plans sync`. Never hand-edit.

Top-level shape: an object with a `project` header and a `plans` array. One entry per plan file (active + shipped). Superseded plans are excluded.

```json
{
  "project": {
    "name": "My Project",
    "description": "One-line project description.",
    "repo": "https://github.com/owner/repo"
  },
  "plans": [
    {
      "file": "FEATURE_NAME.md",
      "title": "Human-readable plan title",
      "status": "active",
      "priority": "P1",
      "owner": "you",
      "type": "feature",
      "depends_on": [],
      "blocks": ["OTHER_PLAN"],
      "last_updated": "2026-05-04",
      "start_date": "2026-05-01",
      "overall": "In progress (Phase 2 of 3)",
      "eta": "2026-05-11",
      "in_flight": true,
      "next_action": "Wire up the state to the API client.",
      "phase_summary": [
        { "phase": 0, "name": "Design",   "status": "shipped",     "date": "2026-04-28" },
        { "phase": 1, "name": "Backend",  "status": "shipped",     "date": "2026-05-01" },
        { "phase": 2, "name": "Frontend", "status": "in_progress", "eta": "2026-05-11"  },
        { "phase": 3, "name": "Tests",    "status": "not_started", "eta": null           }
      ]
    }
  ]
}
```

## project header

| Field | Type | Notes |
|---|---|---|
| `name` | string | Project name. Empty string allowed as a stub; expected to be filled in. |
| `description` | string | One-line description. Empty string allowed as a stub. |
| `repo` | string | Repository URL. Empty string allowed as a stub. |

`/plans sync` preserves the `project` header verbatim across regenerations. If the file is missing the header (legacy array-only file, or first sync of a fresh project), sync creates a stub `{"name":"","description":"","repo":""}` and notes it in the report.

## plans array field sources

| Field | Source |
|---|---|
| `file` | filename |
| `title` | H1 heading in the plan file |
| `status` | frontmatter |
| `priority` | frontmatter |
| `owner` | frontmatter |
| `type` | frontmatter |
| `depends_on` | frontmatter |
| `blocks` | frontmatter |
| `last_updated` | frontmatter |
| `start_date` | frontmatter field if present, else earliest dated phase line, else `last_updated` |
| `overall` | `## Status` banner Overall line |
| `eta` | frontmatter field if present, else latest dated/ETA phase line, else `null` |
| `in_flight` | explicit frontmatter field (`true` = actively being worked now) |
| `next_action` | `## Status` banner Next action line |
| `phase_summary` | per-phase lines in `## Status` banner |

## Phase status values (for phase_summary)

| Banner text | JSON value |
|---|---|
| Done | `shipped` |
| In progress | `in_progress` |
| Not started | `not_started` |
| Paused | `paused` |
| Blocked | `blocked` |
